面向接口编程

  • 完善
    先介绍面向接口编程的使用场景、目的。先了解为什么需要面向接口编程，而不是先定义什么是面向接口编程。

为什么需要面向接口编程

在编程中，有一个原则是DRY——Don't repeat yourself。讲的就是让代码能给尽可能地复用，降低冗余，易于维护。无论是OOP抑或FP中，都提供了一些特性以支持DRY。而面向接口编程，实际上也是在OOP中出于代码复用的目的，给出的一种解决方案。

讲到这里，许多人可能会觉得，如果只是为了代码复用的话，那么直接用继承也可以达到复用的效果，为什么需要接口？下面我们就来看看，只有继承的情况下会有什么问题。借用bird、duck的经典例子，假设已经有了一个bird类：

abstract class Bird {
    abstract void walk();
    abstract void fly();
}

突然有一天，你需要实现一个duck类，出于继承的考量，你很自然的从bird继承并定义了duck类：

class Duck extends Bird {
    public void walk(
        System.out.println("Duck walk");
    )
    
    public void fly();
}

仔细看看上面的fly是一个空方法，因为duck是不会飞的。但是为了符合OOP中的里氏替换原则，我们必须将父类的所有方法实现，所以这里就出现了一个空方法。但是这个没有实际意义的方法，多少会让人有些困扰。

如果想要消除这种看起来不那么合理的设计方案的话，就只能将fly、walk这些行为从bird中抽离出来，定义为一些行为类：

abstract class Walkable {
    abstract void walk();
}

abstract class Flyable {
    abstract void fly();
}

class Duck extends Walkable {
    public void walk(
        System.out.println("Duck walk");
    )
}

这里会带来两个问题：

  1. 在Java这种不支持多继承的语言中，Bird类同时需要fly以及walk两个行为，要如何实现？
  2. 在Duck的继承关系中，继承一个Flyable这样的行为类，似乎与继承的is-a的从属关系不太相符。

所以，Java就给出了接口的概念，以解决这些问题。

接口(interface)的定义

接口是对于一系列行为的约定，它由抽象方法构成，并没有实际的代码。类可以实现多个方法，在类中对接口里定义的抽象方法给出具体的实现。

下面用接口的方式实现一遍：

interface Walkable {
    public void walk();
}

interface Flyable {
    public void fly();
}

class Bird implements Walkable Flyable {
    public void fly ();
    public void walk ();
}

class Duck implements Walkable {
    public void walk();
}

这种方式，能够把类和行为分离，行为作为接口存在，其中的方法在类中予以实现，并不需要继承，消除了is-a的问题。同时，在定义Bird类的时候，我们实现了多个接口，这同时也解决了Java的多继承问题。

面向接口编程与多态

接口除了上面介绍的好处之外，其实还有一个更有威力的用处，那就是面向接口编程。在Java中经常能够听到这个术语，下面我们就继续上面的例子来看看什么情况下需要用到这个技术。

假设现在你需要编写客户端的代码，需要50只duck与50只bird一起执行walk的动作。代码如下：

Duck[] ducks = new Duck[50];
Bird[] birds = new Bird[50];

for (int i = 0; i < 50; i++) {
    ducks[i] = new Duck();
    birds[i] = new Bird();
}

for (Duck d : ducks) {
    d.walk();
}

for (Bird b : birds) {
    b.walk();
}

这样的代码存在着很多的冗余，写起来也不舒服。如果我们要加入多个能walk的动物的话，难道要对每个动物都用for循环来调用walk方法吗。那么我们用面向接口的方式来改写一遍看看：

ArrayList<Walkable> walkers = new ArrayList<>(100);

for (int i = 0; i < 50; i++) {
    walkers.add(new Duck());
    walkers.add(new Bird());
}

for (Walkable walker : walkers) {
    walker.walk();
}

将对象声明为接口类型，调用时，就能够按照接口的特性来调用，不管实际对象到底是哪个类（Duck还是Bird）,就算之后添加新的walk动物的话，调用的代码也不需要修改。这就是面向接口编程的好处。

在上面的这种用法当中，其实是利用了多态的特性。对于不同类的实例对象，调用同样的方法。在维基百科中是这样定义多态的：

多态（英语：polymorphism），是指计算机程序运行时，相同的消息可能会送给多个不同的类别之对象，而系统可依据对象所属类别，引发对应类别的方法，而有不同的行为。

有了这些特性，就使得我们在编写代码的时候，能够做到尽量的遵循DRY原则。同时代码也有了很好的弹性。
